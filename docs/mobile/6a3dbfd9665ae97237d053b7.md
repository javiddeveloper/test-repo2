# [mobile][boilerplate] Configure Compose Dependencies

## Overview

Add Jetpack Compose dependencies to `build.gradle.kts` and enable the Compose build feature.

---

## Changes

### `app/build.gradle.kts`

```kotlin
android {
    buildFeatures {
        compose = true
    }

    composeOptions {
        kotlinCompilerExtensionVersion = "1.5.14"
    }
}

dependencies {
    val composeBom = platform("androidx.compose:compose-bom:2024.06.00")
    implementation(composeBom)

    implementation("androidx.compose.ui:ui")
    implementation("androidx.compose.material3:material3")
    implementation("androidx.compose.ui:ui-tooling-preview")
    debugImplementation("androidx.compose.ui:ui-tooling")

    implementation("androidx.activity:activity-compose:1.9.0")
}
```

---

## Dependency Reference

| Artifact | Purpose |
|---|---|
| `compose-ui` | Core Compose UI primitives and layout |
| `compose-material3` | Material Design 3 components |
| `compose-tooling` / `compose-tooling-preview` | `@Preview` support in Android Studio |
| `activity-compose` | `setContent {}` and `rememberLauncherForActivityResult` |

---

## Notes

- **BOM** (`compose-bom`) aligns all Compose artifact versions automatically — no need to specify individual versions.
- `ui-tooling` is scoped to `debugImplementation` to keep it out of release builds.
- `kotlinCompilerExtensionVersion` must be compatible with the Kotlin version in use; verify against the [Compose to Kotlin compatibility map](https://developer.android.com/jetpack/androidx/releases/compose-kotlin).
- `buildFeatures { compose = true }` is required; without it the Kotlin compiler plugin for Compose will not activate.

---

## Verification

1. Sync Gradle (`File → Sync Project with Gradle Files`).
2. Create a simple composable and annotate it with `@Preview`.
3. Confirm the preview renders in the IDE split view.
4. Run the app and call `setContent { YourComposable() }` from `MainActivity`.
